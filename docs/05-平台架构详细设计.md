# 平台架构详细设计

> 版本：v0.1
> 日期：2026-06-23
> 范围：轨道 A 在线平台 —— Java 平台层 + Python 推理层
> 原则：工程用成熟框架，自己设计架构；训练与在线解耦

---

## 1. 总体架构

```
┌─────────────┐
│  前端 (Web) │  React/Vue —— 提交 prompt、看进度、播放/下载
└──────┬──────┘
       │ HTTP + WebSocket
┌──────▼──────────────────────────────────┐
│  Java 平台层 (Spring Boot 3)              │
│  - REST API / 鉴权 / 用户                 │
│  - 任务管理（创建、状态机、查询）          │
│  - WebSocket 进度推送                      │
│  - 投递任务到队列、监听结果                │
└──────┬───────────────────────────────────┘
       │ RabbitMQ（任务队列）+ Redis（进度/缓存）
┌──────▼───────────────────────────────────┐
│  Python 推理层 (FastAPI + Worker)          │
│  - 消费任务队列                            │
│  - MusicGen / 自研模型（常驻显存）         │
│  - ffmpeg 音频后处理                       │
│  - 写音频到 MinIO，回报进度/结果           │
└──────┬───────────────────────────────────┘
       │
┌──────▼────────┐   ┌──────────────┐
│ MinIO (音频)  │   │ PostgreSQL    │
└───────────────┘   │ (元数据)      │
                    └──────────────┘
```

---

## 2. 组件职责

| 组件 | 技术 | 职责 |
|------|------|------|
| 前端 | React/Vue | 交互、进度展示、音频播放 |
| 平台层 | Spring Boot 3 | 业务逻辑、用户、任务编排、不碰模型 |
| 推理层 | FastAPI + PyTorch | 模型推理、音频处理、GPU 管理 |
| 任务队列 | RabbitMQ | 平台层↔推理层异步解耦、削峰、重试 |
| 缓存/进度 | Redis | 实时进度、结果回传通道、热点缓存 |
| 对象存储 | MinIO | 音频文件（S3 兼容） |
| 元数据库 | PostgreSQL | 用户、任务、生成记录、模型信息 |

**关键原则**：平台层永远不直接调用模型；一切生成请求通过队列异步交给推理层。

## 4. 数据库 Schema（PostgreSQL）

```sql
-- 用户
CREATE TABLE users (
    id           BIGSERIAL PRIMARY KEY,
    username     VARCHAR(64) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    created_at   TIMESTAMPTZ DEFAULT now()
);

-- 生成任务
CREATE TABLE generation_tasks (
    id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id      BIGINT REFERENCES users(id),
    prompt       TEXT NOT NULL,
    model_name   VARCHAR(64) NOT NULL,        -- musicgen-medium / custom-v1
    params       JSONB NOT NULL,              -- duration/temperature/top_k/cfg...
    status       VARCHAR(16) NOT NULL,        -- PENDING/RUNNING/SUCCEEDED/FAILED
    progress     SMALLINT DEFAULT 0,          -- 0-100
    audio_url    TEXT,                        -- MinIO 对象路径
    error_msg    TEXT,
    created_at   TIMESTAMPTZ DEFAULT now(),
    updated_at   TIMESTAMPTZ DEFAULT now()
);
CREATE INDEX idx_tasks_user ON generation_tasks(user_id, created_at DESC);
CREATE INDEX idx_tasks_status ON generation_tasks(status);

-- 模型注册表（支持多模型切换、自研模型接入）
CREATE TABLE models (
    name         VARCHAR(64) PRIMARY KEY,     -- musicgen-medium / custom-v1
    display_name VARCHAR(128),
    type         VARCHAR(32),                 -- builtin / custom
    enabled      BOOLEAN DEFAULT true,
    meta         JSONB                        -- 采样率、最大时长等
);
```

## 5. 任务状态机

```
PENDING ──(worker 取走)──> RUNNING ──(成功)──> SUCCEEDED
   │                          │
   │                          └──(异常)──> FAILED ──(可重试)──> PENDING
   └──(超时未取/取消)──> FAILED
```

- 状态变更同时写 PG（持久）+ Redis（实时推送源）
- 失败任务支持有限次重试（队列层 + 业务层双重）

## 6. 接口定义（平台层 REST）

### 6.1 创建生成任务
```
POST /api/v1/tasks
Body: {
  "prompt": "upbeat electronic dance track",
  "model": "musicgen-medium",
  "params": { "duration": 10, "temperature": 1.0, "top_k": 250, "cfg_coef": 3.0 }
}
Resp: { "taskId": "uuid", "status": "PENDING" }
```

### 6.2 查询任务
```
GET /api/v1/tasks/{taskId}
Resp: { "taskId","status","progress","audioUrl","errorMsg" }
```

### 6.3 任务列表（历史）
```
GET /api/v1/tasks?page=0&size=20
Resp: { "items": [...], "total": N }
```

### 6.4 实时进度（WebSocket）
```
WS /ws/tasks/{taskId}
推送: { "status":"RUNNING", "progress": 45 }
      { "status":"SUCCEEDED", "audioUrl":"..." }
```

### 6.5 模型列表
```
GET /api/v1/models
Resp: [ { "name","displayName","type","meta" } ]
```

## 7. 队列消息协议（平台层 ↔ 推理层）

**任务下发**（平台 → 推理，RabbitMQ queue: `gen.tasks`）：
```json
{
  "taskId": "uuid",
  "prompt": "...",
  "model": "musicgen-medium",
  "params": { "duration": 10, "temperature": 1.0, "top_k": 250, "cfg_coef": 3.0 }
}
```

**进度回报**（推理 → 平台，Redis pub/sub channel: `task.progress`）：
```json
{ "taskId": "uuid", "status": "RUNNING", "progress": 60 }
```

**结果回报**（推理 → 平台）：
```json
{ "taskId": "uuid", "status": "SUCCEEDED", "audioUrl": "minio://bucket/uuid.wav" }
```

平台层订阅 Redis channel，收到后更新 PG 并通过 WebSocket 推给前端。

## 8. 推理层引擎抽象（关键设计：支持自研模型合流）

统一引擎接口，让 MusicGen 和未来自研模型可插拔替换：

```python
# engines/base.py
class MusicEngine:
    def load(self): ...                          # 启动时加载，常驻显存
    def generate(self, prompt: str, params: dict,
                 on_progress) -> bytes: ...       # 返回音频字节，回调进度
    @property
    def sample_rate(self) -> int: ...
```

- `musicgen.py`：包装 audiocraft（轨道 A）
- `custom.py`：轨道 B 自研模型成熟后实现同接口
- worker 按任务的 `model` 字段选择引擎
- **意义**：平台层完全不感知模型实现，换引擎只动推理层

## 11. 设计要点回顾

1. **平台层不碰模型**——纯业务，模型逻辑全在 Python
2. **异步队列解耦**——生成耗时长，必须削峰、可重试、可反馈
3. **模型常驻显存**——worker 启动加载，避免每请求重载
4. **引擎抽象**——为轨道 B 自研模型预留无缝接入点
5. **训练离线解耦**——本文档只管在线；训练见 `06`
