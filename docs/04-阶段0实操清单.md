- conda 或 venv（推荐 conda 管理环境）

## 3. 第一个推理脚本

新建 `generate.py`：

```python
import torchaudio
from audiocraft.models import MusicGen

# 模型规格：small(300M) / medium(1.5B) / large(3.3B)
# 4090 跑 medium 流畅，large 也可（约 10-16GB 显存）
model = MusicGen.get_pretrained('facebook/musicgen-medium')

# 生成参数
model.set_generation_params(
    duration=10,        # 时长（秒）
    temperature=1.0,    # 随机性
    top_k=250,
    top_p=0.0,
    cfg_coef=3.0,       # classifier-free guidance 强度
)

descriptions = [
    "An upbeat electronic dance track with a catchy synth melody",
    "Calm acoustic guitar, relaxing, warm and gentle",
]

wav = model.generate(descriptions)   # 形状: [batch, channels, samples]

sr = model.sample_rate  # 通常 32000
for i, one in enumerate(wav):
    torchaudio.save(f"output_{i}.wav", one.cpu(), sr)
    print(f"saved output_{i}.wav")
```

运行：
```bash
python generate.py
```

**首次运行会自动下载模型权重**（medium 约 6GB），需等待。完成后得到 `output_0.wav`、`output_1.wav`。

## 5. 进阶尝试（建立直觉）

跑通后，玩几个变化加深理解：

1. **改 prompt**：试不同风格、乐器、情绪，观察可控性边界
2. **改 duration**：生成 30s，观察长序列的质量衰减和耗时变化
3. **改 temperature / cfg_coef**：体会随机性和条件遵循度的权衡
4. **换模型规格**：small vs medium vs large，对比质量和显存/速度
5. **旋律条件生成**：MusicGen 支持给定旋律续写，试 `model.generate_with_chroma`
6. **计时**：记录每种规格生成 10s/30s 的耗时，为平台层的超时设计做参考

## 6. 常见问题排查

| 问题 | 原因 | 解决 |
|------|------|------|
| `torch.cuda.is_available()` False | 驱动/CUDA 不匹配 | 重装匹配 CUDA 版本的 torch |
| 下载模型超时 | 网络访问 HuggingFace 慢 | 配置镜像或代理；设 `HF_ENDPOINT` |
| OOM 显存不足 | 模型太大 / duration 太长 | 用更小模型 / 缩短时长 / 减小 batch |
| audiocraft 装不上 | Python 3.12 / 依赖冲突 | 用 Python 3.10，干净环境重装 |
| ffmpeg 报错 | 未安装 ffmpeg | `apt install ffmpeg` |
| 音频有杂音/很差 | 模型固有 / 参数不当 | 调 cfg_coef、temperature；换 medium/large |
